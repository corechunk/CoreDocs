# ⚡ Interrupts & Traps

Interrupts are asynchronous notifications sent by hardware (keyboards, timers, network controllers) to notify the CPU that an event requires immediate attention. Traps (or Exceptions) are synchronous events generated by the CPU itself during execution (e.g. division by zero, page faults).

---

### 🧠 The Interrupt Descriptor Table (IDT)

In x86 architectures, the kernel registers its interrupt handler functions inside a system table called the **Interrupt Descriptor Table (IDT)**.

```text
CPU Hardware Interrupt (Timer IRQ 0)
        |
        v
 [ Locate Vector Index in IDT ] ---> [ Read Gate Descriptor ]
                                              |
                                              v
                                  [ Jump to Handler Address ]
                                  (Executes ISR in Ring 0)
```

The IDT contains $256$ entries (gates). When an interrupt occurs, the CPU:
1.  Stops executing the current program.
2.  Saves the instruction pointer (`EIP`/`RIP`) and CPU flags onto the kernel stack.
3.  Looks up the entry in the IDT matching the interrupt index.
4.  Jumps to the handler address (Interrupt Service Routine - ISR) specified in the gate, transitioning to Ring 0.
